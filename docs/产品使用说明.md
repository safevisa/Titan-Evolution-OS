# Titan Evolution OS — 产品使用说明

本文说明如何**配置、启动与日常使用**本系统。适用于自托管（Docker）与生产前验证环境；域名、HTTPS 与反向代理需您自行按基础设施规范配置。

---

## 1. 架构概览

| 组件 | 说明 |
|------|------|
| **frontend** | Next.js 应用（默认映射 `127.0.0.1:3001`→容器内 3000） |
| **backend** | FastAPI API（默认 `127.0.0.1:8000`） |
| **celery_worker** | 异步执行 Agent 任务（依赖 Redis） |
| **postgres** | 主业务数据库 |
| **redis** | 缓存 / Celery broker |
| **qdrant** | 向量记忆等能力 |

编排文件：仓库根目录 `docker-compose.yml`。

---

## 2. 环境准备

- **Docker Engine** + **Docker Compose** v2（插件 `docker compose`）  
- 可编辑文本编辑器，用于填写环境变量  
- 至少 **一种** 大模型 API 密钥（见下文 `.env`），否则智能任务无法调用 LLM

---

## 3. 首次启动（本地或服务器）

### 3.1 环境变量

在仓库根目录复制示例配置：

```bash
cp .env.example .env
```

**建议至少填写**：

| 变量 | 说明 |
|------|------|
| `POSTGRES_PASSWORD` | 数据库密码（勿使用默认值上线） |
| `AUTH_SECRET` | 可用 `openssl rand -base64 32` 生成 |
| `NEXTAUTH_URL` / `AUTH_URL` | 与浏览器访问的站点根 URL 一致（含协议与域名） |
| `DEEPSEEK_API_KEY` 或 `OPENAI_API_KEY` 等 | 至少配置一项；`LITELLM_DEFAULT_MODEL` 与所用 Key 匹配（见 `.env.example` 注释） |
| `TITAN_ADMIN_EMAILS` | 逗号分隔的管理员邮箱（用于管理端能力） |

**生产建议**：`PUBLIC_API_URL` 留空，由 Nginx 将前端同源的 `/api` 转发到后端（见 `deploy/nginx-tokenply.conf` 示例）。

### 3.2 启动服务

```bash
docker compose build
docker compose up -d
docker compose ps
```

后端容器启动时会执行数据库迁移（如 `alembic upgrade head`，以 `backend/scripts/docker-entrypoint.sh` 为准）。

### 3.3 访问入口

- 前端（本机直连容器映射）：`http://127.0.0.1:3001`  
- API 文档/健康检查（视后端路由）：`http://127.0.0.1:8000`  

公网访问需在主机上配置 **HTTPS + 反向代理** 到上述端口。

## 4. 主要功能页面（前端路由）

路径均带语言前缀 `[locale]`（如 `zh`、`en`），以实际 `frontend/app` 为准：

| 路径（示例） | 用途 |
|--------------|------|
| `/[locale]/dashboard` | 总览 / 仪表盘 |
| `/[locale]/tasks` | 任务发起、历史、协同流水线与执行日志 |
| `/[locale]/agents` | 数字员工（Agent）管理与技能 |
| `/[locale]/skills` | 技能库浏览 |
| `/[locale]/evolution` | 进化与 A/B 等相关能力 |
| `/[locale]/billing` | 用量与计费相关展示 |

| `/login` | 登录入口 |

## 5. 任务与流水线（面向业务用户）

### 5.1 单步任务

在 **任务** 页面选择任务类型、执行 Agent（或智能路由）、填写目标（goal）后执行。执行结果可在历史中查看与评分。

### 5.2 协同流水线（goal pipeline）

选择 **协同流水线** 类型，可按行业插件选择工作流模板；系统将按 DAG 多阶段执行。若某阶段**缺少对应角色的在岗 Agent**，该阶段会被标记为 **skipped**。

### 5.3 缺口闭环（自动）

当流水线成功结束且存在被跳过的阶段时，系统可能自动创建 **`manager_skill_closure`** 类型任务，由管理层 Agent 评估并**生成若干子任务**（具体受租户配置与限额约束）。父流水线输出中可出现 `skill_gap_followup` 字段提示关联的闭环任务 ID。

租户可在 `tenant.config` 中设置 `auto_skill_gap_closure: false` 关闭该行为（若您的版本已实现该开关）。

## 6. 登录与权限

- 支持 **NextAuth v5** 与 **Google OAuth**（可选，见 `.env.example`）。  
- 在 Google Cloud Console 配置 OAuth 回调地址与 `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET`。  
- 管理员名单由 `TITAN_ADMIN_EMAILS` 控制（逗号分隔邮箱）。

## 7. 远程部署（脚本）

仓库提供 `scripts/deploy-to-server.sh`：通过 **SSH + rsync** 将代码同步到服务器 `~/titan-evolution-os-src/` 并在远端执行 `docker compose build && up`。

使用前设置：

```bash
export DEPLOY_HOST=你的服务器IP或域名
export DEPLOY_USER=ubuntu
export DEPLOY_SSH_KEY=/path/to/your.pem
./scripts/deploy-to-server.sh
```

**注意**：不要将私钥 `token.pem` 等提交到 Git；脚本已默认排除同步 `token.pem`。

## 8. 常见问题（FAQ）

**Q：任务一直处于 pending？**  
A：确认 `celery_worker` 容器已启动且无报错；确认 Redis 可用。

**Q：API 返回 401/403？**  
A：检查是否已登录；管理接口是否使用管理员邮箱或正确 API Key（以当前版本实现为准）。

**Q：数据库迁移失败？**  
A：查看 `backend` 容器日志；确认 `DATABASE_URL` 与 Postgres 健康检查通过。

**Q：想换模型？**  
A：修改 `.env` 中 `LITELLM_DEFAULT_MODEL` 及对应厂商的 API Key；重建并重启容器。
