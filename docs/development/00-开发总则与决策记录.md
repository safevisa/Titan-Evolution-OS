## 2. 已冻结决策（ADR）

### ADR-001：Computer Use 部署形态

- **决策**：云沙箱优先；Runner 与 Titan 同 VPC/同机部署；API 进程零 GUI 依赖。
- **理由**：安全隔离、多租户审计、与现有 Docker 生产一致。
- **否决**：客户本机 Connector 作为第一期（列入 Phase 4 可选）。

### ADR-002：Context Sync 第一期数据源

- **决策**：**Gmail + Google Calendar + GitHub 全部接入**。
- **决策**：与 `google_youtube_oauth` **分离 provider**（`google_workspace_oauth`），避免 scope 与 token 混用。
- **决策**：第一期 Gmail **只读**（send 为 M02 第二期）。

### ADR-003：OpenHuman 集成方式

- **决策**：**GPL 合规独立侧车**，与 Titan B2B **并行**；主仓库不 vendor OpenHuman Rust 核心。
- **决策**：Titan 只接收侧车推送的**已压缩摘要**（`sidecar/memory-push`）。
- **否决**：将 OpenHuman 静态链接进 `backend` 镜像。

### ADR-004：执行入口不变

- **决策**：保留并扩展 `execute_capability`；新能力只增 catalog + dispatch 分支。
- **决策**：允许新增顶层包 `context_sync/`、`computer_use/`，避免 `integrations/` 单文件超过 500 行。

### ADR-005：记忆模型

- **决策**：向量仍用 **Qdrant**（现有 `long_term.py`）；层级结构用 **Postgres `memory_tree_nodes`** + payload 关联。
- **决策**：任务 episodic 记忆与 sync 上下文 **分 source 检索**（`task` vs `gmail|gcal|github`）。

### ADR-006：同步周期

- **决策**：Celery Beat **1200 秒（20 分钟）** 与 OpenHuman auto-fetch 对齐。
- **决策**：租户可 `Tenant.config.context_sync.enabled=false` 全关。

### ADR-007：许可

| 组件 | 许可 | 用法 |
|------|------|------|
| Agent-S / gui-agents | Apache-2.0 | Runner 镜像 pip 依赖 |
| OpenHuman | GPL-3.0 | 仅侧车容器 |
| Titan B2B 自研 sync | 项目自有 | backend |

## 4. 术语表

| 术语 | 含义 |
|------|------|
| **Capability** | `catalog.py` 中一条可执行能力，`id` 全局唯一 |
| **Provider** | `integration_connections.provider`，如 `github_oauth` |
| **Grant** | `Tenant.config.integration_grants.enabled_capability_ids` |
| **Control params** | `_idempotency_key`、`_correlation_id`、`_actor`，执行前剥离 |
| **CanonicalDocument** | 同步管道中间态：统一字段后再 chunk |
| **Runner** | `computer-use-runner` 服务，跑 Agent-S3 |
| **侧车** | OpenHuman GPL 独立进程 |
