## 2. 错误码与含义（`execute_capability` 返回）

| error | 含义 | 查什么 |
|-------|------|--------|
| `unknown_capability` | catalog 无此 id | `catalog.py`、拼写 |
| `capability_not_granted` | 白名单未开 | `Tenant.config.integration_grants` |
| `stub_not_implemented` | status=stub | 未完成该模块 PR |
| `missing_connection` | 无 OAuth/凭证 | `integration_connections` |
| `oauth_server_not_configured` | 缺 client_id 等 | `.env` |
| `rate_limited` | Redis 限流 | `capability_metering` |
| `idempotency_replay` | 重复 key | 正常，非 bug |
| `token_expired` | OAuth 刷新失败 | sync_state.last_error、refresh 日志 |
| `concurrency_limit` | Computer Use 满 | `computer_use_runs` count |
| `runner_unreachable` | Runner 挂了 | Runner health、网络 |
| `sidecar_push_denied` | 未开 push | `Tenant.config.sidecar` |

审计表：`capability_audit_logs`（migration 007）。

## 3. Context Sync 排障

### 3.1 从未自动同步

| 检查 | 命令/位置 |
|------|-----------|
| Beat 是否运行 | `docker ps \| grep beat`；无则见 M06 部署 beat |
| Beat 是否注册任务 | `celery -A app.workers.celery_app inspect scheduled` |
| 租户开关 | `SELECT config FROM tenants WHERE id='...'` → `context_sync.enabled` |
| sync_state | `SELECT * FROM integration_sync_states WHERE tenant_id='...'` |
| OAuth 行 | `SELECT provider FROM integration_connections WHERE tenant_id='...'` |

### 3.2 同步失败 token_expired

1. 查 `integration_sync_states.last_error`
2. 查 backend 日志 `google_refresh` / `github`
3. 让用户重新 OAuth 连接
4. 确认 `GOOGLE_WORKSPACE_OAUTH_*` 与 Google Console redirect 一致

### 3.3 Agent 无 synced context

1. Qdrant：`curl http://127.0.0.1:6333/collections` 看 collection 是否存在
2. 是否有 point：`payload.source` 为 gmail/gcal/github
3. `prompt_builder` 是否调用 `fetch_sync_context_block`
4. `inject_max_tokens` 是否 0

### 3.4 Gmail 重复/漏信

- 查 `cursor_json.gmail_history_id`
- 全量修复：清空 cursor + 删 source=gmail nodes + 重跑 `context_sync_run`

## 4. Computer Use 排障

### 4.1 queued 一直不变

- Worker 是否消费 `titan.computer_use.dispatch`
- `COMPUTER_USE_RUNNER_URL` 从 celery 容器内 `curl` 是否通
- Runner 日志是否有 POST `/v1/runs`

### 4.2 failed grounding

- `COMPUTER_USE_GROUND_URL` 可达
- UI-TARS 分辨率参数 1920x1080
- OpenAI/主模型 key 在 Runner env

### 4.3 安全：绝不应出现

- backend 日志 import `pyautogui` → **严重**：违反 ADR-001，回滚部署

## 6. 数据库快速查询

```sql
-- 租户同步状态
SELECT provider, last_success_at, last_error, enabled
FROM integration_sync_states WHERE tenant_id = '<UUID>';

-- 最近 Computer Use
SELECT id, status, step_count, error, created_at
FROM computer_use_runs WHERE tenant_id = '<UUID>'
ORDER BY created_at DESC LIMIT 10;

-- 最近能力审计
SELECT capability_id, ok, error_code, created_at
FROM capability_audit_logs WHERE tenant_id = '<UUID>'
ORDER BY created_at DESC LIMIT 20;
```

## 7. 健康检查脚本（建议加入 `scripts/health-teo-dual.sh`）

```bash
#!/usr/bin/env bash
set -e
curl -sf http://127.0.0.1:8000/docs >/dev/null && echo OK backend
docker ps --format '{{.Names}}' | grep -q celery_beat && echo OK beat || echo FAIL beat
curl -sf http://computer-use-runner:8090/health 2>/dev/null && echo OK runner || echo SKIP runner
```
