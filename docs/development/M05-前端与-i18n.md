# M05 — 前端与 i18n

**前置**：对应后端 API 就绪（可分子模块交付）  
**预估**：与 M02/M03 各 1 周重叠  

---

## 1. 原则

- 所有新文案：`frontend` i18n 资源（`en.json` / `zh.json` 或项目现有结构）
- API 错误码 → 前端映射键 `integrations.errors.{code}`
- 复用 `IntegrationsConsole.tsx` 模式，不新建平行页面除非必要

---

## 2. Context Sync UI

### 2.1 集成页新增卡片

| 卡片 | 操作 |
|------|------|
| Google Workspace | Connect → OAuth redirect；显示 Gmail+Calendar 状态 |
| GitHub | Connect |
| 同步总览 | 上次成功时间、last_error、按钮「立即同步」 |

### 2.2 API 调用

- `GET .../sync-status` 轮询间隔 30s 当 syncing
- `POST .../sync/trigger` 后 toast + 刷新

### 2.3 i18n 键（示例，须完整补全）

```
integrations.contextSync.title
integrations.contextSync.googleWorkspace.description
integrations.contextSync.github.description
integrations.contextSync.lastSync
integrations.contextSync.trigger
integrations.contextSync.status.success
integrations.contextSync.status.error
integrations.contextSync.status.never
```

## 3. Computer Use UI

### 3.2 能力包

integrations.computerUse.runStatus.queued
integrations.computerUse.runStatus.running
