# 外部工具与集成（Agent 现实世界执行层）

本页说明 Titan Evolution OS 如何将 **聊天工具、自媒体、邮件、CRM** 等能力统一进「能力目录」，并在 **租户策略 + 服务端配置 + 用户授权连接** 满足后由 Agent 调用。

## 架构概览

1. **能力目录**（`backend/app/integrations/catalog.py`）  
   每个能力有稳定 `id`（如 `slack_post_message`）、语义版本 `version`（默认 `v1`）、引用形式 `id@v1`（如 `telegram_send_message@v1`）、`category`、`auth_mode`、`status`（`live` / `stub`），以及可选的 `connection_provider_any`。

2. **租户连接**（表 `integration_connections`）  
   - 按 `tenant_id` + `provider` 唯一存储一条连接。  
   - `secret_enc`：Fernet 加密的 JSON（Webhook URL 或 OAuth access/refresh token），**不落明文**。  
   - `meta`：非敏感展示信息（如 Slack team 名、LinkedIn `linkedin_person_urn`）。

3. **加密密钥**（环境变量）  
   - `TITAN_INTEGRATIONS_FERNET_KEY`：主密钥（必填）。  
   - `TITAN_INTEGRATIONS_FERNET_KEY_PREVIOUS`：轮换时旧密钥，用于解密历史数据。  
   生成：`python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"`

4. **OAuth 公网回调**（`TITAN_API_PUBLIC_BASE_URL`）  
   必须为 HTTPS 公网 API 根（无尾斜杠），与各平台开发者后台 redirect URI 一致。

5. **租户策略**（`Tenant.config["integration_grants"]["enabled_capability_ids"]`）  
   - **未配置**：策略上允许目录内能力（执行仍受连接/env 约束）。  
   - **已配置白名单**：仅列表中的 canonical `id` 可执行。  
   - **stub 能力**：需显式勾选；执行返回 `stub_not_implemented`。

6. **统一执行入口**（`execute_capability`）  
   - 解析 `capability_id` 或 `id@v1`（含别名如 `telegram_send` → `telegram_send_message`）。  
   - 按租户 plan 做 **每分钟限流**（Redis `ratelimit:cap:{tenant}:{ref}`）。  
   - **幂等**：`params._idempotency_key` → 表 `capability_idempotency`。  
   - **审计**：表 `capability_audit_logs`。  
   - **计量**：成功后写入 `capability_usage_rollup`（按租户/月/能力 ref 汇总 `bill_unit_usd`）。  
   - HTTP 外呼统一走 **`integration_request`**（`transport.py`：5xx/429 重试、按 provider 熔断）。

7. **Agent 工具闭环**  
   - `list_agent_capabilities` + `capabilities_to_openai_tools` 生成 LLM tools。  
   - `run_agent_with_capability_tools`：`complete_chat_with_tools` 多轮 tool_calls → `execute_capability`。  
   - `GenericRoleAgent` 默认走 tool loop；任务仍支持 `input.invoke_capability` 手动触发。

8. **工作流 DAG**  
   行业插件模板节点可带 `capability_id`（如 hunter + `lead_search` → `apollo_search`）。  
   `GET /api/v1/tasks/workflow-templates` 返回每节点 `role` / `capability_id`。  
   未手写时由 `workflow_capabilities.enrich_dag_config` 按 role+task_type 补全。

## HTTP API（节选）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/integrations/capabilities?tenant_id=` | 目录 + 可执行标志 |
| PATCH | `/api/v1/integrations/tenants/{tenant_id}/grants` | 设置 `enabled_capability_ids` |
| POST | `/api/v1/integrations/tenants/{tenant_id}/grants/apply-pack` | **一键应用能力包**到白名单（`pack_id`, `merge`） |
| POST | `/api/v1/integrations/tenants/{tenant_id}/capabilities/execute` | 执行能力（body: `capability_id`, `params`, 可选幂等/关联字段） |
| GET | `/api/v1/integrations/capability-packs` | 预设能力包列表 |
| GET | `/api/v1/integrations/capability-packs/{pack_id}` | 能力包详情 |
| GET | `/api/v1/integrations/tenants/{tenant_id}/capability-usage` | 当月用量 rollup |
| GET | `/api/v1/integrations/tenants/{tenant_id}/agents/{role}/capabilities` | 角色可见能力 + OpenAI tools |
| GET | `/api/v1/integrations/tenants/{tenant_id}/connections` | 已连接 provider（无密钥） |
| POST | `.../connections/webhook` | Webhook 连接 |
| POST | `.../connections/credential` | 凭证连接（Telegram / WhatsApp / 公众号等） |
| GET | `/api/v1/integrations/oauth/*/start?tenant_id=` | OAuth 授权跳转 |

### 一键应用能力包

```http
POST /api/v1/integrations/tenants/{tenant_id}/grants/apply-pack
Content-Type: application/json

{
  "pack_id": "gtm_outreach",
  "merge": true
}
```

- `merge: true`（默认）：与现有白名单**并集**。  
- `merge: false`：白名单**仅保留**该包内能力。  
- 包内 `capability_refs`（如 `apollo_search@v1`）会解析为 canonical id 写入 grants。

内置包：`gtm_outreach`、`ops_notify`、`social_broadcast`、`hr_finance_notify`（见 `capability_packs.py`）。

### 执行能力（控制字段）

写在 `params` 内、执行前剥离：

| 字段 | 说明 |
|------|------|
| `_idempotency_key` | 幂等键，重复请求返回缓存结果 |
| `_correlation_id` | 审计关联 ID（任务/流水线） |
| `_actor` | 审计主体（如 `agent:{uuid}`） |

限流失败返回：`error: capability_rate_limited`。

## 服务端环境变量（平台应用）

| 变量 | 用途 |
|------|------|
| `TITAN_INTEGRATIONS_FERNET_KEY` | 加密连接密钥（必填） |
| `TITAN_INTEGRATIONS_FERNET_KEY_PREVIOUS` | 密钥轮换解密 |
| `TITAN_API_PUBLIC_BASE_URL` | OAuth 回调公网根 URL |
| `SLACK_*` / `TWITTER_*` / `LINKEDIN_*` 等 | 各平台 OAuth |
| `APOLLO_API_KEY` / `RESEND_API_KEY` | 内置 live 能力 |

## 调用参数（invoke_capability_params）

| 能力 id | 说明 |
|---------|------|
| `discord_post_message` | `content` 或 `text` |
| `slack_post_message` | Webhook：`text`；OAuth：`channel` + `text` |
| `feishu_lark_notify` | `text` |
| `wechat_work_notify` | `markdown` 或 `text` |
| `twitter_x_post` | `text` |
| `linkedin_share` | `text` |
| `telegram_send_message` | `chat_id` + `text` |
| `whatsapp_cloud_text` | `to` + `text` |
| `apollo_search` / `resend_email` | 见 builtin 实现 |

版本化调用示例：`apollo_search@v1`、`telegram_send_message@v1`。

部署：`cd backend && alembic upgrade head`（后端容器 entrypoint 也会自动 upgrade）。

## 前端

- **设置 → 集成**：`/settings/integrations`（`IntegrationsConsole`）  
- **任务中心**：工作流模板展示节点 `capability_id`；能力包参考选择器  

文案使用 i18n key，勿硬编码。

## 多语言

目录 `display_name` / `description` 为中文默认；前端以能力 `id` 为 i18n key。
